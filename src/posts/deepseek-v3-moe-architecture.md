# DeepSeek V3 技术拆解：671B 参数只激活 37B，MoE 架构是怎么做到的

2025 年初，DeepSeek V3 的发布在 AI 圈引发了不小的震动——不是因为它最强，而是因为它太便宜了。

557 万美元，2048 张 GPU。做出来一个在多项基准上超越 GPT-5 的模型。

美国的大厂们花了数亿美元都没做出来的东西，深圳一家公司用不到 600 万美元做到了。这背后的核心技术，就是 **MoE（Mixture of Experts，混合专家）架构**。

---

## 先搞清楚问题：为什么需要 MoE？

传统的 Dense（稠密）神经网络有一个根本矛盾：**想要模型能力强，就要参数多；参数多，推理就慢、成本就高。**

7B 参数能处理日常对话，但写代码差一点；70B 参数写代码不错，但推理一次要耗费大量算力；1000B 参数能力更强，但推理成本高到没法商用。

MoE 的思路是：**不要让所有参数都参与每次计算。** 把参数分组成多个"专家"，每次输入只激活其中几个最相关的专家来处理。

效果是：模型总参数可以很大（容量够），但每次推理只用一小部分（成本低）。

---

## DeepSeek V3 的专家设计

DeepSeek V3 共有 **671B 总参数**，但每次推理只激活 **约 37B 参数**，比例约 5.5%。

它的每个 MoE 层包含两类专家：

### 路由专家（256 个）

每个 token 会从这 256 个路由专家中选出 **8 个** 进行计算。门控网络（Gating Network）负责决定选哪 8 个——它给每个专家打分，取分最高的 8 个。

V3 相比前代（V2）把路由专家数量从约 160 个增加到 256 个，同时把门控函数从 **Softmax 换成了 Sigmoid**。

原因很直接：Softmax 需要对所有专家的分数做归一化，专家数量越多，每个专家分配到的分值越小，选 Top-8 时区分度反而下降。Sigmoid 对每个专家独立打 [0,1] 的分，不受其他专家影响，256 个专家时区分度依然够用。

### 共享专家（1 个）

每个 MoE 层还有 **1 个共享专家**，对所有输入都激活，不参与路由竞争。

它的职责是捕捉通用知识——那些不管什么输入都需要用到的基础能力，比如语言的基本语法结构。让路由专家从这些通用任务中解放出来，专注于处理特定领域的知识。

### 计算流程

```
输入 token
    ↓
Gate Network → 选出 Top-8 路由专家 + 1 个共享专家
    ↓
9 个专家分别处理，产生各自输出
    ↓
按 Gate 权重加权求和 → 当前层输出
```

## 最关键的工程创新：无辅助损失负载均衡

MoE 有一个经典问题：**专家坍塌（Expert Collapse）**。

门控网络会倾向于反复选择同一批表现好的专家，导致其他专家几乎得不到训练，参数白白浪费。为了解决这个问题，训练时通常要加入辅助损失函数（Auxiliary Loss），强迫门控网络均匀地分配流量。

但辅助损失有副作用：它和主任务的损失相互干扰，调不好就会损害模型效果。DeepSeek V1、V2 都用了多个辅助损失来做负载均衡，工程上很复杂。

**V3 把这些辅助损失全部去掉了**，改用一个更优雅的方案：**动态 bias（偏置）**。

做法是给每个专家维护一个 bias 值，当某个专家被选中次数过多时，自动降低它的 bias；被选中少时，自动提高。bias 不参与梯度反传，不影响模型本身的学习，只是在路由时"拨一下指针"。

结果：负载均衡了，模型也没有受到干扰。

## FP8 混合精度训练

训练大模型通常用 BF16（16位浮点）。V3 是行业内首个激进采用 **FP8（8位浮点）** 进行大规模训练的模型。

精度更低的风险是数值不稳定。DeepSeek 团队对哪些层用 FP8、哪些层保持 BF16 做了细致的研究和工程适配，最终实现了在不损失模型质量的前提下，显著降低了训练时的显存占用和带宽压力。

这也是 2048 张 GPU 能完成如此规模训练的重要原因之一。

---

## "自发专化"现象

一个有意思的观察：训练结束后，不同的专家会自发形成不同的"专长"。

- 有的专家专注于语言的句法结构
- 有的专家擅长处理数学符号和推理链
- 有的专家在处理代码逻辑时表现更好
- 有的专家负责捕捉长距离依赖关系

这种专化不是手动设计的，是模型通过端到端训练自己学出来的。这和人类大脑的功能分区有某种相似性——不同区域处理不同类型的信息。

## 我的理解

MoE 不是新技术——Google 2017 年就发了 Sparsely-Gated MoE 的论文，Switch Transformer 是 2021 年的工作。但"会做"和"做好"之间差距很大。

DeepSeek 真正的贡献是大量工程细节：Sigmoid 替代 Softmax 的路由改进、无辅助损失的负载均衡、FP8 训练的稳定性优化……这些都是硬碰硬的工程能力积累。

更重要的是，他们把这些工作都写进了技术报告并开源了。这对整个 AI 社区是真正有价值的贡献——不是营销，是干货。
