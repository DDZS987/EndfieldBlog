# 反代 AI 经验总结：从零开始搭建自己的 API 中转

折腾反代 AI 也有一段时间了，期间踩了很多坑，今天把经验整理成文，希望对有同样需求的朋友有所帮助。

---

## 需要准备哪些工具

在开始之前，先把工具备齐，缺一样都会让后续步骤更麻烦：

| 工具 | 用途 | 备注 |
|------|------|------|
| **指纹浏览器** | 隔离环境，防止指纹污染 | 必须 |
| **家宽 ISP 代理** | 提供干净的住宅 IP | 核心，价格约 1-3 美元/月 |
| **科学上网梯子** | 访问国际网络 | 已有梯子即可 |
| **海外服务器** | 部署反代项目 | 推荐大厂新加坡节点 |
| **反代项目** | 中转 API 请求 | 推荐 sub2api（见首页项目展示） |
| **海外银行卡** | 订阅付费服务 | 国内 Visa/万事达储蓄卡即可 |
| **海外手机号** | 接收验证码 | 非 VOIP 真实手机号更稳妥 |

---

## 关于 IP 质量：这是最关键的一步

所有操作都必须在**指纹浏览器中挂家宽 ISP 代理**下进行。

为什么不能用机场？

机场的机房 IP 是共享的，有大量用户在上面发请求，IP 已经"脏"了。用这类 IP 去注册或操作 AI 服务账号，大概率会触发风控，轻则验证失败，重则 IP + 设备指纹双重封禁。

**购买家宽 ISP 代理后，第一步去 [ping0.cc](https://ping0.cc) 验一下 IP 是否是绿色（住宅属性）**。不绿就直接联系客服，很多服务商会卖垃圾机房 IP 给不检查的用户，价格在 1-3 美元/月的通常是正常范围。

---

## 账号注册

### Google 账号

Google 账号对日常使用的 IP 要求不严，机场 IP 平时用没什么问题。但**注册时对 IP 要求很高**——最好用动态手机 IP 代理或按量计费的家宽 IP，否则容易注册失败或注册后立即封号。

### Claude 账号

比 Google 稍微宽松，但也需要：

1. **家宽 ISP 代理下操作**
2. **海外手机号接码**：`+86`（中国大陆）、`+852`（香港）、`+853`（澳门）均不可用，必须是其他国家的号码

接码方式推荐：
- 自己持有长期使用的海外手机号（最稳定）
- 买断制接码平台，按注册服务类型购买，选真实手机号而非 VOIP 虚拟号

---

## 支付订阅

这块对国内用户是个小门槛，但也不难解决。

**开卡**：招商银行可以直接申请开通海外万事达储蓄卡，其他大部分银行也有类似的外币卡业务，不难开。

**关键点**：Claude 不直接接受国内银行开的卡，需要套一层 **Google Pay**：

1. 把国内万事达卡绑定到 Google Pay
2. 账单地址**不要填中国地址**，填美国任意一个免税州（如俄勒冈、蒙大拿）的地址即可
3. 他们不会联系发卡行核实地址，正常通过后就可以用 Google Pay 内购订阅 Claude Pro

---

## 部署反代服务

账号和订阅搞定后，就可以在服务器上部署反代项目了。

**最容易忽视的一步**：在服务器的反代项目配置里，也要把家宽 ISP 代理填进去，让请求走家宽出口，而不是直接用服务器 IP。

```
用户请求 → 反代服务器 → 家宽ISP代理 → Claude API
```

绝大多数被封的情况都是因为这一步没做——服务器 IP 是机房属性，被风控识别后封禁。

**获取授权 Token**：在指纹浏览器中挂着家宽代理，访问 Claude 授权链接，会直接给出授权 code，填入反代项目即可。Gemini 的 CLI 授权流程同理。

---

## 踩坑总结

整理成一张表，省得大家重走弯路：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 注册时账号立即被封 | IP 不干净/机房 IP | 换家宽 ISP 代理注册 |
| 接码验证失败 | 使用了 VOIP 虚拟号 | 换真实手机号接码平台 |
| 信用卡支付被拒 | 直接绑卡，账单地址填中国 | 套 Google Pay，账单地址填美国免税州 |
| 反代服务频繁封号 | 服务器 IP 是机房属性 | 在反代项目里配置家宽代理出口 |
| IP 买来检测不是绿色 | 服务商卖了机房 IP | 立即联系客服，坚持要住宅属性 IP |

---

## 最后

这套流程走通之后，整体还是比较稳定的，核心思路就是**全程模拟真实住宅用户行为**：真实 IP、真实设备指纹、真实地址。

关键词：指纹浏览器 + 家宽 ISP 代理 + Google Pay 绕支付 + 反代服务配代理出口。
